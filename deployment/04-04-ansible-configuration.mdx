# Ansible Configuration

## Recommended approach

Use the community `community.rabbitmq` Ansible collection for RabbitMQ management:

```bash
ansible-galaxy collection install community.rabbitmq
```

## Typical playbook structure

```yaml
# site.yml
- name: Install and configure RabbitMQ
  hosts: rabbitmq_nodes
  become: true
  roles:
    - role: rabbitmq_install
    - role: rabbitmq_configure
    - role: rabbitmq_cluster

- name: Configure vHosts, users, policies
  hosts: rabbitmq_primary
  become: true
  roles:
    - role: rabbitmq_topology
```

## Key tasks

```yaml
# Install RabbitMQ
- name: Add Team RabbitMQ apt repo
  ansible.builtin.apt_repository:
    repo: "deb https://packagecloud.io/rabbitmq/rabbitmq-server/ubuntu/ {{ ansible_distribution_release }} main"
    state: present
    filename: rabbitmq

- name: Install RabbitMQ
  ansible.builtin.apt:
    name:
      - "erlang-base={{ erlang_version }}"
      - "rabbitmq-server={{ rabbitmq_version }}"
    state: present

# Deploy rabbitmq.conf from template
- name: Deploy rabbitmq.conf
  ansible.builtin.template:
    src: rabbitmq.conf.j2
    dest: /etc/rabbitmq/rabbitmq.conf
    owner: rabbitmq
    group: rabbitmq
    mode: '0640'
  notify: restart rabbitmq

# Create vHost
- name: Create vHost
  community.rabbitmq.rabbitmq_vhost:
    name: "{{ item }}"
    state: present
  loop: "{{ rabbitmq_vhosts }}"

# Create user
- name: Create service user
  community.rabbitmq.rabbitmq_user:
    user: "{{ item.name }}"
    password: "{{ item.password }}"
    vhost: "{{ item.vhost }}"
    configure_priv: "{{ item.configure | default('') }}"
    write_priv: "{{ item.write | default('') }}"
    read_priv: "{{ item.read | default('') }}"
    state: present
  loop: "{{ rabbitmq_users }}"
  no_log: true

# Set policy
- name: Apply delivery-limit policy
  community.rabbitmq.rabbitmq_policy:
    name: delivery-limit
    vhost: /
    pattern: ".*"
    definition:
      delivery-limit: 3
    apply_to: quorum-queues
    state: present
```

## AceMQ Take

- Store `rabbitmq_users` passwords in Ansible Vault or retrieve from a secrets manager in the playbook.
- Version-pin both Erlang and RabbitMQ in your inventory variables. Ansible will not prevent unintended version drift without explicit pinning.
- Run the topology playbook (`vhosts`, `users`, `policies`) idempotently after every cluster change to drift-correct.
