# Secrets Management

## What to protect

| Secret | Risk if exposed |
|---|---|
| RabbitMQ admin password | Full cluster control |
| Service account passwords | Queue access, potential message exfiltration |
| TLS private keys | Man-in-the-middle attacks |
| Erlang cookie | Arbitrary cluster command execution |
| Management API credentials | Full API access |

## Kubernetes: Secret resources

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: rabbitmq-credentials
type: Opaque
data:
  username: <base64>
  password: <base64>
```

Mount into pods as environment variables or files:

```yaml
env:
- name: RABBITMQ_PASSWORD
  valueFrom:
    secretKeyRef:
      name: rabbitmq-credentials
      key: password
```

## HashiCorp Vault integration

```bash
# Store RabbitMQ credentials in Vault
vault kv put secret/rabbitmq/production \
  username=app-svc \
  password=<generated-password>

# Retrieve in application startup
RABBITMQ_PASSWORD=$(vault kv get -field=password secret/rabbitmq/production)
```

For dynamic credentials, use Vault's RabbitMQ secrets engine, which generates short-lived users:

```bash
vault secrets enable rabbitmq
vault write rabbitmq/config/connection \
  connection_uri="https://rabbitmq.internal:15671" \
  username=vault-admin \
  password=<admin-password>
```

## Erlang cookie

```bash
# Generate a strong random cookie
openssl rand -hex 32 > /var/lib/rabbitmq/.erlang.cookie
chmod 400 /var/lib/rabbitmq/.erlang.cookie
chown rabbitmq:rabbitmq /var/lib/rabbitmq/.erlang.cookie
```

Store the cookie value in Vault/Kubernetes Secret and inject at node startup. Never hardcode in config files.

## TLS certificate rotation

```bash
# Check certificate expiry
openssl x509 -enddate -noout -in /etc/rabbitmq/certs/server.crt

# Rotate: deliver new cert, then reload
cp new-server.crt /etc/rabbitmq/certs/server.crt
rabbitmqctl eval 'ssl:clear_pem_cache().'
# Or: rolling restart for full cert reload
```

## AceMQ Take

- Never store RabbitMQ passwords in `rabbitmq.conf` or Ansible inventory in plaintext. Always use a secrets manager.
- Use Vault's RabbitMQ secrets engine for service accounts in high-security environments. Short-lived credentials limit blast radius.
- Rotate the Erlang cookie annually or after any suspected compromise. Rotation requires a cluster restart.
