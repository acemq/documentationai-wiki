🐇 **Upstream reference:** [Erlang Version Requirements](https://www.rabbitmq.com/docs/which-erlang)

RabbitMQ runs on the Erlang/OTP runtime. Each RabbitMQ version requires a specific Erlang version range. Running with an unsupported Erlang version is not recommended and may cause unexpected behavior.

## Version Compatibility Matrix (Recent)

| RabbitMQ Version | Min Erlang | Max Erlang | Notes |
|---|---|---|---|
| 4.1.x | 27.x | latest | |
| 4.0.x | 26.2 | 27.x | |
| 3.13.x | 26.0 | 26.x | End of community support |
| 3.12.x | 25.0 | 26.x | End of life |

> Always check [https://www.rabbitmq.com/docs/which-erlang](https://www.rabbitmq.com/docs/which-erlang) for the current matrix before upgrading.

## Installing Erlang

RabbitMQ recommends using either:
- **Erlang from Cloudsmith** — `apt.rabbitmq.com` or `rpm.rabbitmq.com` repositories (includes RabbitMQ-specific patches)
- **Erlang packages from Erlang Solutions** — `packages.erlang-solutions.com`

Avoid OS-bundled Erlang (e.g. `apt install erlang` on Ubuntu) — these are often older versions not in the supported range.

```bash
# Check current Erlang version
rabbitmqctl eval 'erlang:system_info(otp_release).'
# or
erl -eval 'erlang:display(erlang:system_info(otp_release)), halt().' -noshell
```

## Upgrading Erlang

Erlang can be upgraded independently of RabbitMQ during a rolling upgrade (stop node, upgrade Erlang, upgrade RabbitMQ, start node). Combined Erlang + RabbitMQ upgrades on the same node are supported when both versions are in the compatibility matrix.

## Erlang and TLS

RabbitMQ's TLS capabilities (cipher suites, protocol versions) depend on the Erlang SSL application version. Newer Erlang versions support TLS 1.3 and stronger cipher suites. If you need TLS 1.3, ensure your Erlang version includes `crypto` with OpenSSL 1.1.1+.

## AceMQ Take

**Use Cloudsmith Erlang packages.** The packages at `apt.rabbitmq.com` are specifically packaged and tested with RabbitMQ. OS-bundled or self-compiled Erlang versions introduce variables that make support and troubleshooting harder.

**Pin the Erlang version in your IaC.** Your Ansible playbooks, Dockerfiles, or Helm charts should specify an exact Erlang version. Floating on `latest` means an Erlang upgrade can happen automatically and take your RabbitMQ out of the supported range.

**Include Erlang version in your upgrade runbook.** When upgrading RabbitMQ, the Erlang version to target is a required input. Document it explicitly in your runbook — don't discover it during the maintenance window.
