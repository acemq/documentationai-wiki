# Networking

## Port reference

| Port | Protocol | Use |
|---|---|---|
| 5672 | AMQP 0-9-1, 1.0 | Client connections (plaintext) |
| 5671 | AMQP/TLS | Client connections (TLS) |
| 15672 | HTTP | Management UI and HTTP API |
| 15671 | HTTPS | Management UI (TLS) |
| 1883 | MQTT | MQTT clients |
| 8883 | MQTT/TLS | MQTT clients (TLS) |
| 61613 | STOMP | STOMP clients |
| 61614 | STOMP/TLS | STOMP clients (TLS) |
| 5552 | Stream | RabbitMQ stream protocol |
| 25672 | Erlang dist | Inter-node communication |
| 4369 | EPMD | Node discovery |

## Listener configuration

```ini
# rabbitmq.conf

# Bind to specific interface only
listeners.tcp.default = 0.0.0.0:5672

# TLS listener
listeners.ssl.default = 5671
ssl_options.certfile = /etc/rabbitmq/certs/server.crt
ssl_options.keyfile  = /etc/rabbitmq/certs/server.key
ssl_options.cacertfile = /etc/rabbitmq/certs/ca.crt
ssl_options.verify = verify_peer
ssl_options.fail_if_no_peer_cert = true

# Disable plaintext if TLS is enforced
listeners.tcp = none
```

## TCP connection tuning

```ini
# rabbitmq.conf

# AMQP heartbeat (seconds)
heartbeat = 60

# TCP backlog
tcp_listen_options.backlog = 4096

# TCP keepalive
tcp_listen_options.keepalive = true
tcp_listen_options.linger.on = true
tcp_listen_options.linger.timeout = 0
```

## Connection limits

```ini
# Maximum total connections (across all protocols)
connection_max = 10000
```

Each AMQP connection consumes ~50–200 KB RAM. 10,000 connections = 0.5–2 GB RAM just for connection state.

## AceMQ Take

- Always enable TLS for client connections in production. Use TLS 1.2 minimum; prefer TLS 1.3.
- Disable the management UI plaintext port (15672) in production — expose only 15671 behind an authenticated reverse proxy.
- Set `heartbeat = 60` for most clients. Lower values (10–30s) detect dead connections faster but increase heartbeat overhead under load.
