# Troubleshooting Guide

## Diagnostic first steps

For any RabbitMQ issue, run these in order:

```bash
# 1. Is the node alive?
rabbitmq-diagnostics ping

# 2. Any alarms active?
rabbitmqctl list_alarms

# 3. Cluster state
rabbitmqctl cluster_status

# 4. Node status summary
rabbitmq-diagnostics status

# 5. Check queues with problems
rabbitmqctl list_queues name messages consumers state --formatter pretty_table
```

| Symptom | Section |

## Diagnostic data collection

When opening a support ticket or escalating an incident:

```bash
# Collect full diagnostic bundle
rabbitmq-diagnostics collect_and_upload_diagnostics

# Or manually:
rabbitmq-diagnostics status > /tmp/rabbitmq-status.txt 2>&1
rabbitmqctl cluster_status >> /tmp/rabbitmq-status.txt 2>&1
rabbitmqctl list_queues name messages consumers state >> /tmp/rabbitmq-status.txt 2>&1
rabbitmqctl list_connections name state >> /tmp/rabbitmq-status.txt 2>&1
rabbitmq-diagnostics memory_breakdown >> /tmp/rabbitmq-status.txt 2>&1
tar czf /tmp/rabbitmq-diagnostics.tar.gz /tmp/rabbitmq-status.txt /var/log/rabbitmq/
```

## Log analysis

```bash
# Recent errors
grep -E '(error|warning)' /var/log/rabbitmq/rabbit.log | tail -50

# Connection events
grep 'connection' /var/log/rabbitmq/rabbit.log | tail -20

# Memory events
grep 'memory' /var/log/rabbitmq/rabbit.log | tail -10
```
