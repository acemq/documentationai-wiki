## Breaking changes summary

| Version | Breaking change | Migration required |
|---|---|---|
| 4.0 | Classic mirrored queues removed | Yes: migrate to quorum queues |
| 4.0 | Default queue type changed to quorum | Yes: audit queue declarations |
| 4.1 | Khepri default (Mnesia deprecated) | No: automatic |
| 4.3 | Mnesia removed | Yes: must be on Khepri |
| 4.3 | Partition handling strategies removed | Yes: remove config keys |

## AceMQ Take

- Always check the Erlang version requirements before upgrading RabbitMQ. Installing a new RabbitMQ version without upgrading Erlang is a common failure mode.
- The AMQP 1.0 client compatibility row is important for .NET shops: the `RabbitMQ.Client` 7.x+ supports AMQP 1.0 natively; older versions do not.
- The feature matrix `4.3` column is a forward reference for planning: if you're on 4.x now, be aware that Mnesia and partition handling strategies will be removed in 4.3.
